# `/xhs:archive`

Use this command to close a workflow and write lessons back to the repo.

## Read

- brief
- draft
- review
- optional outcome notes
- relevant knowledge files
- `.xhsspec/prompts/archive.md`

## Write

- `retrospective.md`
- knowledge updates

## Writeback Protocol

1. Write `retrospective.md` first.
2. Then write one structured lesson block to the correct knowledge file.
3. Keep the retrospective lightweight, but include workflow, outcome, and publish package when available.
4. Distinguish reusable patterns from one-off observations.
5. Suggest repo/spec updates only when they are concrete.
6. Use `<placeholder>...</placeholder>` for any still-unresolved lesson.

## Response Shape

1. What worked
2. What failed
3. What to reuse
4. What to avoid
5. Which knowledge file to update
6. Whether a publish package was linked
