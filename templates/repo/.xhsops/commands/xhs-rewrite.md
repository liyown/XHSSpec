# `/xhs:rewrite`

Use this command after review feedback to produce the next draft version.

## Read

- latest draft
- latest review
- `.xhsops/brand/*`
- `.xhsops/specs/note.spec.md`
- `.xhsops/specs/creation.spec.md`
- `.xhsops/prompts/rewrite.md`

## Write

- next draft artifact

## Writeback Protocol

1. Use the review as the change list.
2. Do not preserve weak wording just to stay close to the old draft.
3. Keep frontmatter and artifact identity intact.
4. If a change trades off another quality, say so briefly.

## Response Shape

1. What changed
2. Revised hook
3. Revised body
4. Revised CTA
5. Recommended next step
