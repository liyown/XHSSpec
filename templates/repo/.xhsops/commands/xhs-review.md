# `/xhs:review`

Use this command to critique a draft against repo rules.

## Read

- draft artifact
- `.xhsops/brand/tone.md`
- `.xhsops/brand/taboo.md`
- `.xhsops/specs/review.spec.md`
- `.xhsops/specs/creation.spec.md`
- `.xhsops/prompts/review.md`

## Write

- `review.md` or campaign note review file

## Writeback Protocol

1. Lead with verdict.
2. Prioritize issues by impact, not by section order.
3. Name what should be preserved in the next draft.
4. Make rewrite guidance actionable enough for `/xhs:rewrite`.
5. Use `<placeholder>...</placeholder>` for any still-unresolved judgment.

## Response Shape

1. Verdict
2. Highest-priority fixes
3. Rewrite guidance
4. Recommended next step
