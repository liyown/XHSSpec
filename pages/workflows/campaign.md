# Campaign 工作流

**一句话：做一个系列的内容，或者规划一个月的更新节奏时，用这个。**

---

## 什么时候该用这个

你想做一个系列的内容。

比如：

- 「这个月我要做 AI 效率工具专题」
- 「下周我要发三篇连更」
- 「我想测试三个不同的角度，看哪个效果更好」
- 「这一季的主题是『程序员成长』，我想做 5 篇成一个系列」

这些时刻的共同点是：**你不是在写一篇，而是在写多篇。**

你需要的不是「怎么写一篇」，而是「怎么管理一个系列」——篇与篇之间怎么串联、整体节奏怎么安排、每一篇的角度怎么差异化。

**Campaign 工作流就是帮你完成这个「系统性创作」的过程。**

它不是帮你写一篇稿子，而是帮你管理一个创作项目。

### 适合 Campaign 的场景

- 做一个系列的内容（3篇以上）
- 规划一个月/一季度的更新节奏
- 想测试不同角度，看哪个效果更好
- 周计划、月度主题、季度规划
- 需要多篇内容互相配合（比如引流款、转化款、品牌款）

### 不适合 Campaign 的场景

- 临时起意想写一篇 → 用 Quick
- 热点来了想快速响应 → 用 Trend
- 只有一个想法，没想清楚要做几篇 → 先用 Quick

简单来说：**当你脑子里有一个「我要做 X 篇」的规划时，那就是 Campaign。**

---

## Campaign 和 Quick 的区别

| 维度 | Quick | Campaign |
|------|-------|----------|
| 创作规模 | 单篇 | 多篇（3篇以上） |
| 管理方式 | 一个 run | 一个 campaign + 多个 note |
| 规划深度 | 选题级 | 系列级 |
| 适合场景 | 临时起意、日常更新 | 系统性规划 |
| 复杂度 | 低 | 中 |
| 典型案例 | 突然想写一篇 | 做一个「AI 效率工具」系列 |

Quick 是一次性创作，Campaign 是系统性创作。

---

## 一个完整的创作故事

让我带你走一遍 Campaign 的完整流程，看看它是怎么管理一个 3 篇系列的。

### 第一步：规划

周一的早上，你坐在咖啡店里，想一个问题：二月要做什么内容？

你有两个选择：

1. **佛系更新**：想到什么写什么，发一篇是一篇
2. **系列化运营**：做一个主题，让它产生累积效应

你选择了第二种。你决定这个月做一个「AI 效率工具」的系列内容，目标是增长粉丝。

你想发 3 篇，每篇的角度不同：

1. 第一篇：钩子要强，容易传播——「5个让我效率翻倍的AI神器」
2. 第二篇：方法要具体，建立专业度——「我是怎么把AI整合进工作流的」
3. 第三篇：案例或转化承接——「用AI做副业，我的第一个订单是怎么来的」

你打开 Claude Code，直接说：

```
/xhs:plan

主题：AI 效率工具
目标：增长粉丝
篇数：3篇
周期：weekly
```

AI 没有立刻开始写，而是先跟你确认了几个问题：

- 每篇的目标受众有没有细分？
- 三篇之间的逻辑关系是什么？（并列 / 递进 / 悬念）
- 发布节奏是什么？（连发 / 隔天发 / 一周一篇）

你回答完之后，AI 说：「好，我帮你创建 Campaign。」

### 第二步：生成 proposal

AI 读取了你的 brand 配置，生成了 **proposal.md**：

```
# Campaign Proposal: AI 效率工具系列

## 背景
目标：通过3篇系列内容，建立「AI效率博主」的专业形象，带动粉丝增长

## 目标受众
- 程序员群体
- 产品经理
- 对效率工具有兴趣的职场人士

## 系列结构
- 第1篇：认知建立——「5个神器」
- 第2篇：方法论——「工作流整合」
- 第3篇：转化——「实战案例」

## 预期效果
- 第1篇目标：收藏+转发（传播向）
- 第2篇目标：评论+关注（建立信任）
- 第3篇目标：私信+转化（变现向）
```

你看完觉得方向没问题。AI 继续生成了 **brief.md**（campaign 级别的整体方向）和 **tasks.md**（任务清单，3 篇笔记的任务分配）。

这里的 `tasks.md` 现在不只是 checklist，它会分成两层：

- **Note Strategy**：每一篇在系列里的角色、要验证的角度、发布目标
- **Execution Board**：每一篇当前做到 draft / review / rewrite 哪一步

### 第三步：第一篇：钩子

你说：「先写第一篇 note-01。」

AI 生成了初稿。你审核后，觉得开头还不够抓人。审核意见：

- 开头太长了，应该直接上工具名
- 5个工具的介绍可以更简洁，每个人用一句话说清楚
- CTA 可以更明确：「收藏起来慢慢看」

你说：「帮我改写开头，要更有冲击力。」

改写完再审核，通过。

### 第四步：第二篇：方法论

继续写第二篇 note-02。

你说：「写第二篇，方法论的角度。」

AI 生成了初稿。你审核后发现，这篇是方法论，需要更扎实。审核意见：

- 工作流的描述可以更具体，最好有一个实际案例
- 缺少一个「before vs after」的对比
- 中间有一段太干了，可以加一个场景

你把这些意见告诉 AI，让它改写。改完你觉得 OK，通过。

### 第五步：第三篇：转化

继续写第三篇 note-03。

你说：「写第三篇，转化篇，要放 CTA。」

AI 生成了初稿。你审核后发现，这篇需要有明确的行动引导。审核意见：

- CTA 可以更强，比如引导私信
- 案例部分可以更真实一点
- 结尾可以加一个「下期预告」

改完之后你觉得 OK，通过。

### 第六步：发布

三篇都审核完了，现在是「ready」状态。你说：「帮我生成发布包。」

AI 生成了一套发布包，包含 3 篇笔记的：

- 正文
- 首屏策略（3个不同的风格）
- 图文拆解和配图建议
- demo.html（3组可以直接截图的演示稿）
- posting-guide.md（3篇的发布 checklist）

你把整个目录交给运营同事：「按这个包发。」

### 第七步：归档

最后，你说：「归档这个 campaign。」

AI 把整个创作过程归档，并生成了这个 campaign 的复盘：

- 选题：AI效率工具系列（3篇）
- proposal 执行情况：完全按计划执行
- 每篇的审核意见和改写轮次
- 哪些地方做得好、哪些角度测试失败了下次可以改进

这些经验会沉淀到你的知识库里。下次你再做类似主题的时候，可以直接从知识库里调取：「上次做这个系列，效果不错，这次可以换个角度试试。」

## 交互方式

```
/xhs:plan                            # 发起 campaign
/xhs:draft note-01                   # 写第一篇
/xhs:review note-01                  # 审核第一篇
/xhs:rewrite note-01                 # 改写第一篇（如需要）
/xhs:draft note-02                   # 写第二篇
/xhs:review note-02                  # 审核第二篇
/xhs:rewrite note-02                 # 改写第二篇（如需要）
/xhs:draft note-03                   # 写第三篇
/xhs:review note-03                  # 审核第三篇
/xhs:rewrite note-03                 # 改写第三篇（如需要）
/xhs:publish                         # 批量发布
/xhs:archive                         # 归档，复盘经验
/xhs:status                          # 查看 campaign 状态
```

## Campaign 里的层级关系

Campaign 跟 Quick 不一样，它是「容器」，里面有层级：

| 层级 | 文档 | 作用 |
|------|------|------|
| Campaign 级 | proposal、brief、tasks | 整体规划、系列结构、任务分配 |
| Note 级 | draft、review、rewrite | 单篇笔记的创作和审核 |

所以你可以把 Campaign 理解成一个“系列看板”：

- `proposal.md` 和 `brief.md` 决定这个系列到底想讲什么
- `tasks.md` 决定每一篇在系列里扮演什么角色
- `drafts/` 和 `reviews/` 决定每一篇当前做到哪了
- `status --target <campaign-id>` 会直接告诉你当前推荐推进哪一篇 note
- 它还会解释为什么当前最该推进这一篇，而不是只给你一个 note ID
- 它还会告诉你当前优先动作是 draft、review 还是 publish
- 它还会给出一条可直接复制给 Claude / Codex 的下一步话术
- `publish/` 会把每一篇已发布 note 的进度反映回 series board
- `status --target <campaign-id>` 还会显示整个系列的发布时间线，方便你看节奏和完成度

所以 campaign 管理的是「多个 note 怎么串联」，不是「一篇 note 怎么写」。

---

## 状态流转

```
created → planned → briefing → drafting → reviewing → iterating → ready → archived
```

| 状态 | 意味着什么 |
|------|------------|
| `created` | campaign 已创建，等待 agent 补 proposal、brief、tasks |
| `planned` | proposal 已生成，规划已完成 |
| `briefing` | campaign brief 和 tasks 正在形成 |
| `drafting` | 至少有一篇 note 进入写作 |
| `reviewing` | 系列正在推进中（至少一篇 note 已审核或已发布，但系列还没全部完成） |
| `iterating` | 改写中（至少一篇 note 在改写） |
| `ready` | 计划内 note 都已经发布完，整个系列可以归档 |
| `archived` | campaign 经验已经沉淀 |

## 什么情况下选 Campaign 而不是 Quick

| 场景 | 推荐工作流 |
|------|------------|
| 突然有个想法想记录 | Quick |
| 日常更新，没有系列规划 | Quick |
| 想做一个系列内容（3篇以上） | Campaign |
| 想规划一个月的更新 | Campaign |
| 想测试不同角度 | Campaign |
| 周计划、月度主题 | Campaign |

简单来说：**如果你脑子里已经有一个「我要做 X 篇」的规划，那就是 Campaign。**

## 为什么 Campaign 值得用

很多人觉得「写一篇就够了为什么要搞这么复杂」。

让我告诉你为什么：

### 1. 系列的累积效应 > 单篇的随机爆

一篇笔记爆了，然后呢？没有了。

一个系列做下来，每一篇都在为账号建立资产。读者看完第一篇觉得不错，会关注你等第二篇；看完第二篇觉得专业，会转发给同事；看完第三篇觉得有用，会私信你。

这就是系列的威力。

### 2. 系统性规划让创作更轻松

很多人创作痛苦的根源是「每天都要想今天发什么」。

Campaign 帮你一次性把一个月的选题想清楚。你只需要在月初规划一次，然后按计划执行就行。

### 3. 数据回流让下次更好

Campaign 结束后，所有的创作经验都会沉淀到知识库里。

- 哪篇效果最好？为什么？
- 哪个角度失败了？为什么？
- 受众更喜欢什么样的内容？

下次再做 Campaign 的时候，你可以直接从知识库里调取这些经验，而不是从零开始。

---

## Campaign 的最佳实践

1. **先想清楚系列结构**：是并列、递进还是悬念？
2. **每篇要有差异化**：不要三篇都是同一个调性
3. **设置不同的目标**：有的负责传播，有的负责建立信任，有的负责转化
4. **做好数据追踪**：每篇发布后，记录数据，下 campaign 复盘时用
5. **及时归档**：每次 campaign 结束后都要归档，这是你最重要的知识资产
