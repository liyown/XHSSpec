# Trend 工作流

**一句话：热点来了，先判断该不该跟，再决定怎么跟。**

---

## 什么时候该用这个

你正在刷微博，看到「OpenAI 发布了新模型」上了热搜。

你正在刷小红书，看到一个话题突然爆了。

你正在看新闻，看到某个事件在朋友圈刷屏了。

这些时刻的共同点是：**热点来了，而且来得很急。**

热点之所以叫热点，就是它来得快、去得也快。你需要在短时间内判断：

- 这个热点跟我的账号定位搭不搭？
- 我能从什么角度切入？
- 到底值不值得跟？
- 强行蹭的话，风险是什么？

**Trend 工作流就是帮你完成这个「快速判断 + 快速产出」的过程。**

它不是让你盲目追热点，而是让你在热潮中保持冷静——先分析适切性，再决定要不要动手。

### 适合 Trend 的场景

- 热搜上出现跟你领域相关的热点
- 行业里有了新动态，你想快速响应
- 某个事件在社交媒体上刷屏了
- 你想测试一下某个角度有没有人看

### 不适合 Trend 的场景

- 你已经有明确的系列计划 → 用 Campaign
- 你只是想记录一个想法 → 用 Quick
- 这个热点跟你八竿子打不着 → 别跟，浪费时间

## 一个完整的创作故事

让我带你走一遍 Trend 的完整流程。

### 第一步：发现热点

早上九点，你刷微博，看到「OpenAI 发布了新模型」上了热搜。你是一个写 AI 效率方向的博主，这个热点必须得蹭。

但你没有立刻开始写。因为你被热点伤过——之前有一次你追了一个热点，写了两个小时，结果发出去没人看，还被读者评论说「硬蹭」。

你决定用工作流来说话。

### 第二步：发起热点

你打开 Claude Code，说：

```
/xhs:hot

主题：OpenAI 新模型发布
来源：发布会摘要
紧急度：high
```

AI 没有立刻开始写稿，而是先问了你几个问题：

- 这个热点你打算从哪个角度切入？（技术解读 / 效率提升 / 普通人怎么用）
- 你的目标受众是谁？（技术人员 / 产品经理 / 普通用户）
- 你想达到什么效果？（展示专业度 / 引流 / 纯蹭热度）

你回答完之后，AI 说：「好，我帮你创建 Trend 工作流。」

### 第三步：fit-check（最关键的环节）

AI 读取了你的 brand 配置，生成了两个关键文档：

- **trend-brief.md**：这个热点的基本信息——发生了什么、为什么重要
- **fit-check.md**：适切性判断——这个热点跟你的账号搭不搭

你打开 fit-check 仔细看：

```
## 适切性分析

### 相关性评估
- 领域相关度：高
- 受众匹配度：高
- 时间窗口：短（48小时内最佳）

### 风险提示
- 发布会信息铺天盖地，你的角度需要差异化
- 已经有大量同类内容产出，你需要找到独特切入点

### 建议角度
1. 「普通人怎么用这个新模型提升效率」——差异化角度，避开技术解读的红海
2. 「这个新功能让我的一天发生了什么变化」——场景化，更容易引发共鸣
3. 「和上一代对比，这些变化值得你升级吗」——对比测评，实用导向

### 结论
建议跟，但角度要选好，不要硬蹭技术细节
```

你觉得这个分析很在理。特别是「差异化角度」那部分，点醒了你——如果你也去写技术解读，肯定拼不过那些专业AI媒体。

于是你说：

```
/xhs:fit approved
```

热点通过了 fit-check，现在可以进入正稿阶段。

### 第四步：创作

你说：「继续写这篇，角度选第一个——普通人怎么用新模型提升效率。」

AI 生成了初稿。你快速审核了一下，发现开头太平淡了，于是说：

```
/xhs:review
```

审核意见出来了：

- 开头太长了，应该直接切入场景
- 「效率提升」这个词太抽象，应该用具体例子
- CTA 要明确：引导读者去评论区分享他们的用法

你把这些意见告诉 AI，让它改写。改完之后你看了一遍，觉得 OK 了。

### 第五步：发布

你说：「帮我生成发布包。」

AI 生成了一套完整的发布包。封面的方向是「效率党必看」，配图建议是做一张新旧模型对比的信息卡——这正好是你擅长做的内容类型。

你说：「归档这次创作。」

AI 把整个过程归档，并生成了复盘：

- 选题来源：热点（OpenAI新模型）
- fit-check 结果：通过
- 审核意见：3条
- 改写轮次：1轮
- 耗时：约40分钟

整个过程，从看到热点到发布包生成，不到一小时。

## 另一个故事：热点被放弃

热点不是每次都要跟。让我给你讲一个「放弃」的故事。

### 第一步：发现热点

周五下午，团队讨论要不要跟一个热点：「某明星官宣新恋情」。

团队的运营同事很兴奋：「这个热点流量超大，我们必须蹭一下！」

你觉得有点奇怪：我们是写 AI 效率的账号啊，跟明星恋情有什么关系？

但你又说不过他们。运营同事的理由是：「流量大啊，不蹭白不蹭。」

你说：「这样吧，我们用工作流来分析一下，让数据说话。」

### 第二步：发起热点

你说：

```
/xhs:hot

主题：某明星官宣新恋情
来源：微博热搜
紧急度：medium
```

AI 生成了 fit-check.md。

你打开一看：

```
## 适切性分析

### 相关性评估
- 领域相关度：极低
- 受众匹配度：低（我们的受众是程序员和效率爱好者）
- 时间窗口：中等

### 风险提示
- 强行跟热点会显得刻意
- 可能会伤害品牌调性——读者会疑惑「这个账号怎么开始追星了」
- 就算流量高，转化率也会很低

### 建议
放弃。或者，如果非要跟，可以用一种完全不同的方式（比如「从程序员角度看这个公告的语法错误」），但风险较大

### 结论
不建议跟，建议放弃
```

### 第三步：放弃

你把这个分析给团队看了。

运营同事看完之后说：「好吧，确实不太搭。万一读者取关就亏了。」

于是你说：

```
/xhs:fit rejected
/xhs:archive --outcome dropped
```

这个决策过程被完整记录了下来。下次再有人提议跟热点，直接调出这份记录：「上次分析过了，不适合。」

这就是 Trend 工作流的价值：**它不是阻止你追热点，而是让你在追热点之前，先冷静三秒钟。**

---

## 交互方式

```
/xhs:hot                              # 发起热点
/xhs:fit approved                     # 适切性通过
/xhs:fit rejected                     # 适切性拒绝
/xhs:review                           # 审核草稿
/xhs:rewrite                          # 改写（如需要）
/xhs:publish                          # 生成发布包
/xhs:archive --outcome dropped       # 归档（放弃热点）
```

## 状态流转

### 热点通过

```
created → fit-checking → fit-approved → drafting → reviewed → done → archived
```

| 状态 | 意味着什么 |
|------|------------|
| `created` | 热点已创建，等待 agent 补完 trend-brief 和 fit-check |
| `fit-checking` | 正在判断热点与品牌是否匹配 |
| `fit-approved` | 允许进入草稿阶段 |
| `drafting` | 初稿正在撰写 |
| `reviewed` | 审核意见已经给出 |
| `done` | 发布包已经生成 |
| `archived` | 经验已经沉淀到知识库 |

### 热点放弃

```
created → fit-checking → fit-rejected → archived
```

| 状态 | 意味着什么 |
|------|------------|
| `created` | 热点已创建 |
| `fit-checking` | 正在判断适切性 |
| `fit-rejected` | 热点被放弃 |
| `archived` | 决策过程已归档，经验沉淀 |

## 为什么 fit-check 不能省

我知道你在想什么：「不就是追个热点吗，有必要这么复杂吗？」

让我告诉你为什么。

### 热点是最容易让人冲动的场景

> 「先写了再说，管它搭不搭呢。」

这是很多运营人员追热点时的心声。然后呢？

- 写了两小时，发出去没人看
- 流量很高，但转化率为零
- 读者开始疑惑：「这个账号到底是干嘛的」
- 甚至被老读者取关：「你怎么变成这样了」

热点之所以危险，是因为它**自带流量，让人看不清风险**。

### fit-check 强制你停下来想三秒

这四个问题必须想清楚：

1. **跟我的品牌有没有关系？** —— 不要强行关联
2. **对我的受众有没有价值？** —— 不要自嗨
3. **有没有合规风险？** —— 不要踩红线
4. **我能不能写出差异化角度？** —— 不要硬蹭

这四个问题想清楚了，后续的创作效率才会高。

### 放弃也是好决策

记住：**不是每个热点都值得跟。**

放弃一个热点，不是失败，而是理性的选择。Trend 工作流帮你记录了这个决策过程，让你可以理直气壮地说：「我们分析过了，不适合。」

## Quick vs Trend：什么时候该用哪个

| 场景 | 推荐工作流 |
|------|------------|
| 突然有个想法想记录 | Quick |
| 日常更新，没有系列规划 | Quick |
| 热点来了，想快速响应 | Trend |
| 想测试某个角度有没有人看 | Trend |
| 已经有明确的系列计划 | Campaign |

简单来说：

- **Quick** = 你有一个想法
- **Trend** = 热点来了，你要判断该不该跟
- **Campaign** = 你有一个计划
